# Manual Setup Guide & Database Schema (Custom `dropout` Schema)

Follow these instructions to configure the Supabase database under a dedicated `dropout` schema, set up Row Level Security (RLS) policies, define environment variables, and start the FastAPI ML prediction backend.

---

## 1. Environment Variables Configuration

Create or update your `.env` (or `.env.local` for Next.js) in the root directory with the following variables:

```env
# Supabase API Credentials
NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.your-anon-key

# External FastAPI Prediction Model Endpoint
BACKEND_URL=http://localhost:8000
```

---

## 2. FastAPI Backend Integration Setup

We have created a dedicated FastAPI backend inside the `backend/` directory.

### A. Place ML Pickle Files
Place your trained Machine Learning Scikit-learn pickle files directly inside the `backend/` directory:
- `dropout_model.pkl` (The trained Random Forest / Scikit-learn Classifier)
- `scaler.pkl` (The Scikit-learn StandardScaler)
- `label_encoder.pkl` (The Scikit-learn LabelEncoder for categorical variables)

*Note: If these files are not found or fail to load, the backend will gracefully fall back to a deterministic mathematical warning calculation to prevent server crashes.*

### B. Install Dependencies
Open your terminal inside the `backend/` directory and install the required Python packages:
```bash
pip install fastapi uvicorn scikit-learn==1.6.1 pandas joblib pydantic
```

### C. Start the FastAPI Server
Launch the API using `uvicorn` in your terminal:
```bash
cd backend
uvicorn main:app --reload --port 8000
```
The FastAPI documentation will be available at `http://127.0.0.1:8000/docs`.

## 3. Supabase Client Configuration (Next.js Side)

Because you are using a custom schema named `dropout` instead of the default `public` schema, configure your Supabase client in `src/lib/supabase.ts` to look for tables inside the `dropout` schema:

```typescript
import { createClient } from "@supabase/supabase-js";

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL || "https://your-supabase-project.supabase.co";
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY || "your-supabase-anon-key";

// Initialize client with the custom schema option
export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  db: {
    schema: "dropout"
  }
});
```

## 4. Supabase SQL Database Schema

Run the following SQL script in your Supabase SQL Editor. It creates the `dropout` schema and places all lookup tables, logs, grades, triggers, and indices within it.

```sql
-- Create custom schema
CREATE SCHEMA IF NOT EXISTS dropout;

-- Set search path for this execution session
SET search_path TO dropout, public;

-- 1. Departments Table
CREATE TABLE dropout.departments (
    id SERIAL PRIMARY KEY,
    department_name TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 2. Teachers Table
-- Matches the user ID generated by Supabase Auth (auth.users).
CREATE TABLE dropout.teachers (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 2.5. Administrators Table
-- Matches the user ID generated by Supabase Auth (auth.users).
CREATE TABLE dropout.administrators (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 3. Classes Table
CREATE TABLE dropout.classes (
    id SERIAL PRIMARY KEY,
    class_name TEXT NOT NULL,
    department_id INTEGER NOT NULL REFERENCES dropout.departments(id) ON DELETE CASCADE,
    teacher_id UUID REFERENCES dropout.teachers(id) ON DELETE SET NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 4. Students Table (Refined for ML Prediction Pipeline)
CREATE TABLE dropout.students (
    student_id TEXT PRIMARY KEY, 
    name TEXT NOT NULL,
    age INTEGER NOT NULL CHECK (age > 0),
    gender TEXT NOT NULL CHECK (gender IN ('Male', 'Female')),
    family_income NUMERIC(10,2) NOT NULL,
    internet_access BOOLEAN DEFAULT TRUE NOT NULL,
    study_hours_per_day NUMERIC(4,1) DEFAULT 2.0 NOT NULL CHECK (study_hours_per_day >= 0.0),
    attendance_rate NUMERIC(5,2) DEFAULT 100.00 NOT NULL CHECK (attendance_rate BETWEEN 0.00 AND 100.00),
    assignment_delay_days INTEGER DEFAULT 0 NOT NULL,
    travel_time_minutes INTEGER DEFAULT 0 NOT NULL,
    part_time_job BOOLEAN DEFAULT FALSE NOT NULL,
    has_scholarship BOOLEAN DEFAULT FALSE NOT NULL,
    gpa NUMERIC(3,2) DEFAULT 4.00 NOT NULL CHECK (gpa BETWEEN 0.00 AND 4.00),
    semester_gpa NUMERIC(3,2) DEFAULT 4.00 NOT NULL CHECK (semester_gpa BETWEEN 0.00 AND 4.00),
    cgpa NUMERIC(3,2) DEFAULT 4.00 NOT NULL CHECK (cgpa BETWEEN 0.00 AND 4.00),
    financial_problem BOOLEAN DEFAULT FALSE NOT NULL,
    semester_year INTEGER NOT NULL CHECK (semester_year BETWEEN 1 AND 4),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 5. Class Students Junction Table (Roster Link)
CREATE TABLE dropout.class_students (
    id SERIAL PRIMARY KEY,
    class_id INTEGER NOT NULL REFERENCES dropout.classes(id) ON DELETE CASCADE,
    student_id TEXT NOT NULL REFERENCES dropout.students(student_id) ON DELETE CASCADE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
    UNIQUE (class_id, student_id)
);

-- 6. Attendance Logs Table
CREATE TABLE dropout.attendance_logs (
    id SERIAL PRIMARY KEY,
    student_id TEXT NOT NULL REFERENCES dropout.students(student_id) ON DELETE CASCADE,
    class_id INTEGER NOT NULL REFERENCES dropout.classes(id) ON DELETE CASCADE,
    date DATE NOT NULL,
    status TEXT NOT NULL CHECK (status IN ('Present', 'Absent')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
    UNIQUE (student_id, class_id, date)
);

-- 7. Grades Table
CREATE TABLE dropout.grades (
    id SERIAL PRIMARY KEY,
    student_id TEXT NOT NULL REFERENCES dropout.students(student_id) ON DELETE CASCADE,
    class_id INTEGER NOT NULL REFERENCES dropout.classes(id) ON DELETE CASCADE,
    assignment_1 NUMERIC(4,2) DEFAULT 0.00 NOT NULL CHECK (assignment_1 BETWEEN 0.00 AND 20.00),
    midterm NUMERIC(4,2) DEFAULT 0.00 NOT NULL CHECK (midterm BETWEEN 0.00 AND 30.00),
    final_exam NUMERIC(5,2) DEFAULT 0.00 NOT NULL CHECK (final_exam BETWEEN 0.00 AND 50.00),
    final_gpa NUMERIC(3,2) DEFAULT 4.00 NOT NULL CHECK (final_gpa BETWEEN 0.00 AND 4.00),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
    UNIQUE (student_id, class_id)
);

## 5. Row Level Security (RLS) Policies

Enable RLS and define policies inside the custom `dropout` schema.

```sql
-- Enable Row Level Security on all schema tables
ALTER TABLE dropout.departments ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.teachers ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.administrators ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.classes ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.students ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.class_students ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.attendance_logs ENABLE ROW LEVEL SECURITY;
ALTER TABLE dropout.grades ENABLE ROW LEVEL SECURITY;

-- Helper Function: Check if the authenticated user is an Administrator
CREATE OR REPLACE FUNCTION dropout.is_admin()
RETURNS BOOLEAN AS $$
BEGIN
  RETURN EXISTS (SELECT 1 FROM dropout.administrators WHERE id = auth.uid());
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- ==========================================
-- A. Policies for DEPARTMENTS
-- ==========================================
CREATE POLICY "Admins have full access on departments" ON dropout.departments
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view departments" ON dropout.departments
    FOR SELECT TO authenticated USING (NOT dropout.is_admin());

-- ==========================================
-- B. Policies for TEACHERS
-- ==========================================
CREATE POLICY "Admins have full access on teachers" ON dropout.teachers
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view their own profile" ON dropout.teachers
    FOR SELECT TO authenticated USING (auth.uid() = id);

-- ==========================================
-- B.2. Policies for ADMINISTRATORS
-- ==========================================
CREATE POLICY "Admins have full access on administrators" ON dropout.administrators
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

-- ==========================================
-- C. Policies for CLASSES
-- ==========================================
CREATE POLICY "Admins have full access on classes" ON dropout.classes
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view their assigned classes" ON dropout.classes
    FOR SELECT TO authenticated USING (teacher_id = auth.uid());

-- ==========================================
-- D. Policies for STUDENTS
-- ==========================================
CREATE POLICY "Admins have full access on students" ON dropout.students
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view student names/IDs" ON dropout.students
    FOR SELECT TO authenticated USING (
        EXISTS (
            SELECT 1 FROM dropout.class_students cs
            JOIN dropout.classes c ON cs.class_id = c.id
            WHERE cs.student_id = students.student_id AND c.teacher_id = auth.uid()
        )
    );

-- ==========================================
-- E. Policies for CLASS_STUDENTS (Junction)
-- ==========================================
CREATE POLICY "Admins have full access on class_students" ON dropout.class_students
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view rosters for their classes" ON dropout.class_students
    FOR SELECT TO authenticated USING (
        EXISTS (
            SELECT 1 FROM dropout.classes c
            WHERE c.id = class_students.class_id AND c.teacher_id = auth.uid()
        )
    );

-- ==========================================
-- F. Policies for ATTENDANCE_LOGS
-- ==========================================
CREATE POLICY "Admins have full access on attendance_logs" ON dropout.attendance_logs
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view/insert/update attendance for their classes" ON dropout.attendance_logs
    FOR ALL TO authenticated USING (
        EXISTS (
            SELECT 1 FROM dropout.classes c
            WHERE c.id = attendance_logs.class_id AND c.teacher_id = auth.uid()
        )
    ) WITH CHECK (
        EXISTS (
            SELECT 1 FROM dropout.classes c
            WHERE c.id = attendance_logs.class_id AND c.teacher_id = auth.uid()
        )
    );

-- ==========================================
-- G. Policies for GRADES
-- ==========================================
CREATE POLICY "Admins have full access on grades" ON dropout.grades
    FOR ALL TO authenticated USING (dropout.is_admin()) WITH CHECK (dropout.is_admin());

CREATE POLICY "Teachers can view/insert/update grades for their classes" ON dropout.grades
    FOR ALL TO authenticated USING (
        EXISTS (
            SELECT 1 FROM dropout.classes c
            WHERE c.id = grades.class_id AND c.teacher_id = auth.uid()
        )
    ) WITH CHECK (
        EXISTS (
            SELECT 1 FROM dropout.classes c
            WHERE c.id = grades.class_id AND c.teacher_id = auth.uid()
        )
    );
```

## 5. Automatic Aggregations (Database Triggers)

Execute these statements to add triggers that update aggregate metrics in `dropout.students` automatically.

```sql
-- A. Trigger to recalculate Student Attendance Rate
CREATE OR REPLACE FUNCTION dropout.update_student_attendance_rate()
RETURNS TRIGGER AS $$
DECLARE
    total_days INT;
    present_days INT;
    new_rate NUMERIC(5,2);
BEGIN
    SELECT COUNT(*) INTO total_days FROM dropout.attendance_logs WHERE student_id = NEW.student_id;
    SELECT COUNT(*) INTO present_days FROM dropout.attendance_logs WHERE student_id = NEW.student_id AND status = 'Present';
    
    IF total_days > 0 THEN
        new_rate := (present_days::NUMERIC / total_days::NUMERIC) * 100.00;
    ELSE
        new_rate := 100.00;
    END IF;
    
    UPDATE dropout.students SET attendance_rate = new_rate WHERE student_id = NEW.student_id;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_attendance_change
AFTER INSERT OR UPDATE OR DELETE ON dropout.attendance_logs
FOR EACH ROW EXECUTE FUNCTION dropout.update_student_attendance_rate();


-- B. Trigger to recalculate Student GPA
CREATE OR REPLACE FUNCTION dropout.update_student_gpa()
RETURNS TRIGGER AS $$
DECLARE
    avg_gpa NUMERIC(3,2);
BEGIN
    SELECT AVG(final_gpa) INTO avg_gpa FROM dropout.grades WHERE student_id = NEW.student_id;
    
    IF avg_gpa IS NOT NULL THEN
        UPDATE dropout.students SET gpa = avg_gpa WHERE student_id = NEW.student_id;
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_grades_change
AFTER INSERT OR UPDATE OR DELETE ON dropout.grades
FOR EACH ROW EXECUTE FUNCTION dropout.update_student_gpa();
```
